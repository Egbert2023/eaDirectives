Overview of the provided directives

  • eaMaskCode with eaMask-html|css|json|js - The directive is an aid to displaying code as developers expect it to. You can put the e.g. Html code 'any-html-code' into the tag.
  • eaAccording - The accordion function is supported with two tags. (<ea-acc-coat .../> and <ea-acc-key .../>)
  • eaPathLink - This directive is at the top of every html page and represents the link to the current page and its parent page.

    e.g. The SiteMap page
    Service SiteMap
  • eaNavi - Provides the complete navigation with the file 'naviList.json' and creates the navigation menu. Alternatively, a SiteMap can be generated, which is used for display in the application.
  • eaAddHtml - The reloading of Html files is required for the separation of application and content.
  • eaLoadJson - Reads all JSON files and stores them in Javascript objects. This means that they are available for the application at any time.
  • eaImg und eaImgBox - Display of images or groups of images (see imgBoxList.json) in their own Html files and their full-screen display.
  • eaNews - The reloading of Html files is required for the separation of application and content.
  • eaVideo - This directive simplifies the integration of small videos that are stored together with your own content.
  • eaCookies - This directive supports the management of cookies. The entire content of the cookie banner is stored in parameters in the JSON file 'paramsApp.json'. The use of cookies can be switched off with the parameter (data-cookies-used="false"). (As in this application)
  • eaFooter - Displays the Footer.html as a fixed footer.
